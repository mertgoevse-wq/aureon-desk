## Root Causes Found & Fixed

### 1. Synchronous Port Probe Failure (Windows)
**File:** `src/main/services/live-preview.service.ts`

The `findAvailablePort()` function spawned child processes via execSync with inline JavaScript.
On Windows this failed due to quote escaping and Electron policy, causing port collisions.

**Fix:** Replaced with async in-process net.createServer() socket binding check.

### 2. Blank Preview Frame During starting State
**File:** `src/renderer/src/pages/LivePreview.tsx`

The iframe was only mounted when status === running. During starting the panel showed idle state.

**Fix:** Render iframe in starting AND running. Loading overlay shown when starting.

### 3. Iframe Caching Stale Pages
Added key={status.id} to <iframe> so React fully remounts it on each new session.

### 4. Status Race Condition (Demotion)
Pipeline step events could overwrite running back to starting. Guard added: if prev.status
is running or error, only URL is updated — status is preserved.

### 5. Hidden URL Input Not Reflecting Live URL
[data-testid="preview-url-input"] value bound to status.url || customUrl || "".

### 6. Async/Sync IPC Type Mismatch
preview:createDemo IPC handler updated to async returning Promise<CodingDemoResult>.
